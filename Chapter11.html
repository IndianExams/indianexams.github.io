Best Practices & Patterns

Vue 3's Composition API gives more flexibility, but with flexibility comes responsibility 😅. This chapter shows how to organize, mix, and optimize your Vue code.

1. Organizing Code with Composition API

Instead of putting all logic in one setup(), you can split it into composables (useXyz functions). This makes logic reusable and maintainable.

Example: useCounter composable

// composables/useCounter.js
import { ref } from "vue"

export function useCounter(initial = 0) {
  const count = ref(initial)
  const increment = () => count.value++
  const decrement = () => count.value--
  return { count, increment, decrement }
}

Usage in a component

<script setup>
import { useCounter } from "@/composables/useCounter"

const { count, increment, decrement } = useCounter(5)
</script>

<template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="increment">+</button>
    <button @click="decrement">-</button>
  </div>
</template>

Benefit

Code is cleaner, logic is testable, and multiple components can reuse it.

2. Mixing Options API & Composition API

Vue 3 allows hybrid components: you can use Options API (data, methods, computed) together with Composition API (setup()).

This is useful for:

  • Migrating from Vue 2 → Vue 3 gradually.
  • Working in large teams where not everyone is comfortable with Composition API.

Example: Mixed usage

<script>
import { ref, onMounted } from "vue"

export default {
  data() {
    return {
      message: "Hello from Options API"
    }
  },
  setup() {
    const count = ref(0)
    onMounted(() => console.log("Mounted with Composition API"))
    return { count }
  },
  methods: {
    increment() {
      this.count++   // still works fine
    }
  }
}
</script>

<template>
  <div>
    <p>{{ message }}</p>
    <p>Count: {{ count }}</p>
    <button @click="increment">Increment</button>
  </div>
</template>

Note

You can read Composition API state in Options methods via this.

3. Performance Considerations

Vue 3 is already optimized, but there are best practices:

1. Use ref / reactive carefully

  • Too many deeply nested reactive objects = overhead.
  • Prefer shallowRef / shallowReactive when deep reactivity isn't needed.

2. Avoid unnecessary reactivity

// ❌ Inefficient: makes whole object reactive
const state = reactive({ items: [] })

// ✅ Better: use ref for array
const items = ref([])

3. Use v-memo or v-once for static content

<h1 v-once>Static Title</h1>

4. Lazy-load components with defineAsyncComponent

import { defineAsyncComponent } from "vue"
const AsyncModal = defineAsyncComponent(() => import("./Modal.vue"))

5. WatchEffect vs Watch

  • Use watchEffect for auto-tracking,
  • Use watch when you need specific dependencies.

✅ Summary

  • Organize: Use composables (useXyz) to keep logic modular.
  • Mixing APIs: Safe to mix Options & Composition APIs → useful in migrations.
  • Performance: Optimize reactivity, use async/lazy components, and avoid unnecessary reactive wrapping.
