Reusable Composables

Reusable Composables

Composables are reusable functions that encapsulate stateful logic using Vue's reactivity system. They make your code modular, testable, and maintainable.

1. Creating a Simple Composable

// useCounter.js
import { ref } from "vue"

export function useCounter(initialValue = 0) {
  const count = ref(initialValue)

  function increment() {
    count.value++
  }
  function decrement() {
    count.value--
  }

  return {
    count,
    increment,
    decrement
  }
}

Usage:

<script setup>
import { useCounter } from "./useCounter"

const { count, increment, decrement } = useCounter(5)
</script>

<template>
  <p>Count: {{ count }}</p>
  <button @click="increment">+</button>
  <button @click="decrement">-</button>
</template>

2. Sharing Logic Across Components

You can import the same composable in multiple components and share its logic.

<script setup>
import { useCounter } from "./useCounter"

const { count, increment } = useCounter(0)
</script>

<template>
  <p>Shared count: {{ count }}</p>
  <button @click="increment">Increment</button>
</template>

Each call to useCounter() creates an independent instance of state.

3. Using Lifecycle Hooks in Composables

Composables can also use Vue lifecycle hooks.

// useMouse.js
import { ref, onMounted, onUnmounted } from "vue"

export function useMouse() {
  const x = ref(0)
  const y = ref(0)

  function update(e) {
    x.value = e.pageX
    y.value = e.pageY
  }

  onMounted(() => window.addEventListener("mousemove", update))
  onUnmounted(() => window.removeEventListener("mousemove", update))

  return { x, y }
}

Usage:

<script setup>
import { useMouse } from "./useMouse"
const { x, y } = useMouse()
</script>

<template>
  <p>Mouse position: {{ x }}, {{ y }}</p>
</template>

4. Composables with Dependencies

You can pass arguments to composables to configure their behavior.

// useFetch.js
import { ref, onMounted } from "vue"

export function useFetch(url) {
  const data = ref(null)
  const error = ref(null)

  onMounted(async () => {
    try {
      const res = await fetch(url)
      data.value = await res.json()
    } catch (err) {
      error.value = err
    }
  })

  return { data, error }
}

Usage:

<script setup>
import { useFetch } from "./useFetch"

const { data, error } = useFetch("https://jsonplaceholder.typicode.com/posts")
</script>

<template>
  <div v-if="error">Error: {{ error.message }}</div>
  <ul v-else>
    <li v-for="post in data" :key="post.id">{{ post.title }}</li>
  </ul>
</template>

5. Best Practices for Composables

  • Use the use prefix (e.g., useCounter, useFetch)
  • Keep composables stateless where possible
  • Return only what's needed (refs, reactives, functions)
  • Reuse across multiple components
  • Can be organized in a /composables folder

Note

With composables, Vue 3 encourages a hook-like pattern (similar to React hooks) but with more flexibility.

✅ Key Takeaways

  • Composables are reusable functions that encapsulate stateful logic
  • They make code more modular, testable, and maintainable
  • Each composable call creates an independent instance of state
  • Composables can use Vue lifecycle hooks and accept parameters
  • Follow best practices like using the "use" prefix and keeping them stateless
