Vue 3 Composition API Cheatsheet – Chapter 1

Introduction to Composition API

What is the Composition API?

The Composition API is a new way to write Vue components, introduced in Vue 3, that makes code more organized, reusable, and scalable compared to the Options API.

  • Options API → organizes code by option type (data, methods, computed, etc.).
  • Composition API → organizes code by logic and features, making it easier to reuse.

The setup() Function

  • The entry point of the Composition API inside a component.
  • Runs before lifecycle hooks.
  • Returns values that can be used inside the template.
<script setup>
import { ref } from "vue"

const message = ref("Hello Vue 3!")
</script>

<template>
  <h1>{{ message }}</h1>
</template>

ref() vs reactive()

  • ref() → For primitive values (string, number, boolean).
  • reactive() → For objects & arrays.

Important difference

With ref, you must use .value in JavaScript. With reactive, you do not use .value (Vue unwraps it automatically).

<script setup>
import { ref, reactive } from "vue"

// ref example
const count = ref(0)
function increment() {
  count.value++  // must use .value
}

// reactive example
const user = reactive({
  name: "Daniel",
  age: 34
})
function birthday() {
  user.age++  // no .value needed
}
</script>

<template>
  <button @click="increment">Count: {{ count }}</button>
  <p>{{ user.name }} - {{ user.age }}</p>
  <button @click="birthday">Increase Age</button>
</template>

✅ Key Takeaways

  • Use setup() to define state & logic.
  • Use ref() for primitive values, reactive() for objects.
  • ref → requires .value in JS; reactive → no .value needed.
  • Both are automatically unwrapped inside templates.
