Vue 3 Composition API Cheatsheet – Chapter 2

Computed & Watchers

computed()

  • Used for derived state (values based on other reactive data).
  • Automatically updates when dependencies change.
  • Caches the value until one of the dependencies changes.
<script setup>
import { ref, computed } from "vue"

const firstName = ref("Daniel")
const lastName = ref("Smith")

// computed property
const fullName = computed(() => `${firstName.value} ${lastName.value}`)
</script>

<template>
  <p>{{ fullName }}</p>
</template>

Key point

Use computed when you need caching → it only recalculates when dependencies change.

Great for expensive operations.

watch()

  • Reacts to specific reactive data and executes a function when the value changes.
  • Deep watching available with the deep: true option.
<script setup>
import { ref, watch } from "vue"

const count = ref(0)

watch(count, (newVal, oldVal) => {
  console.log(`Count changed from ${oldVal} → ${newVal}`)
})
</script>

<template>
  <button @click="count++">Count: {{ count }}</button>
</template>

watchEffect()

  • Runs immediately and re-runs whenever any reactive dependency inside changes.
  • No need to declare explicit dependencies.
  • Avoid reading/writing to values like loading.value that could cause infinite loops.
<script setup>
import { ref, watchEffect } from "vue"

const price = ref(100)
const tax = ref(0.18)

watchEffect(() => {
  console.log("Total:", price.value * (1 + tax.value))
})
</script>

⚠️ Infinite Loop Warning

If you read a value inside watchEffect and also mutate it inside the effect, it can trigger an infinite loop.

<script setup>
import { ref, watchEffect } from "vue"

const loading = ref(false)

watchEffect(() => {
  console.log("loading:", loading.value) // 👈 dependency is read here
  loading.value = true  // 👈 we mutate here
  // This causes an infinite loop! 
  loading.value = false
})
</script>

Important

watchEffect runs again whenever reactive data that is read inside it changes.

Writes (mutations) do not trigger reactivity directly. But reading and then writing to a value inside the same effect can cause infinite re-runs.

✅ Key Takeaways

  • computed → for derived state (cached, declarative).
  • watch → for side effects when specific data changes.
  • watchEffect → for auto-tracking dependencies without listing them.
  • Avoid reading and writing the same value inside a watchEffect to prevent infinite loops.
