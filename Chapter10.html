Render Functions & JSX with Composition API

Render Functions & JSX

Vue's template syntax covers 90% of cases, but sometimes we need programmatic rendering. That's where render functions or JSX come in.

1. Basics of Render Functions

Vue provides a h() function (short for "hyperscript") to manually describe what the component should render. This can be useful when you want dynamic structure that's hard to express in templates.

Example: Render function in Composition API

<script setup>
import { h } from "vue"

const RenderFunctionComponent = () => {
  return h("div", { class: "box" }, [
    h("h2", "Hello from Render Function!"),
    h("p", "This was rendered without a template.")
  ])
}

export default RenderFunctionComponent
</script>

Explanation

  • h("div", props, children)
  • Children can be strings or nested h() calls.
  • No <template> is needed.

2. JSX Usage in Vue 3

Instead of h() nesting, we can use JSX (JavaScript XML-like syntax). Vue 3 supports JSX with the right Babel plugin (@vitejs/plugin-vue-jsx or Vue CLI JSX support).

Example: JSX in Vue 3

<script setup lang="jsx">
const name = "Vue 3 + JSX"

export default () => {
  return (
    <div class="box">
      <h2>Hello from JSX!</h2>
      <p>{name}</p>
    </div>
  )
}
</script>

Explanation

  • JSX lets us write markup-like syntax inside JS.
  • {} is used to insert variables.
  • Looks like React JSX, but it's still Vue (with reactivity, refs, etc.).

3. When to Use Render Functions or JSX?

Use them when:

  • You need programmatic control over the structure.
  • You're writing a component library (slots, dynamic children).
  • You want conditional elements that are too complex for v-if/v-for.

Avoid them if:

  • A normal <template> is enough — templates are usually clearer.
  • Your team isn't familiar with JSX or render functions.
  • You don't need the extra flexibility they provide.

✅ Summary

  • h() → manual render function.
  • JSX → more natural syntax for writing render logic.
  • Both integrate fully with Composition API (you can use ref, computed, etc.).
  • Use when you need programmatic control over rendering structure.
