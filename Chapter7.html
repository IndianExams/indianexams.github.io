Advanced Reactivity

Advanced Reactivity

Vue 3 provides powerful reactivity features. In addition to basic ref and reactive, there are advanced techniques that provide more control over reactivity.

shallowRef

  • Similar to ref, but only makes the top-level properties reactive.
  • Nested properties will not be reactive.
  • Useful when working with large objects to avoid deep reactivity.
import { shallowRef } from "vue"

const user = shallowRef({
  name: "John",
  address: { city: "New York", zip: "10001" }
})

user.value.address.city = "San Francisco"  // This won't trigger reactivity

toRefs

  • Converts a reactive object into a ref for each property.
  • Useful when passing a reactive object into a function/component while maintaining reactivity.
import { reactive, toRefs } from "vue"

const state = reactive({
  name: "John",
  age: 30
})

const { name, age } = toRefs(state) // Converts properties to refs

readonly

  • Makes a reactive object or ref read-only.
  • It prevents accidental mutations, useful for immutable states.
import { reactive, readonly } from "vue"

const state = reactive({
  name: "John",
  age: 30
})

const readonlyState = readonly(state)

readonlyState.name = "Jane"  // Error: Cannot mutate a readonly object

customRef

  • Creates a custom ref with more control over get and set operations.
  • You can create refs that have custom behavior (e.g., delayed updates, computed refs with caching, etc.).
import { customRef } from "vue"

function useCustomRef(initialValue) {
  return customRef((track, trigger) => {
    let value = initialValue

    return {
      get() {
        track()   // tell Vue: this value was read
        return value
      },
      set(newValue) {
        value = newValue
        trigger() // tell Vue: this value changed
      }
    }
  })
}

const myRef = useCustomRef(0)

console.log(myRef.value) // track() runs → dependency collected
myRef.value = 5          // trigger() runs → watchers re-run

track and trigger Explanation

  • track(): Tells Vue that the value has been read, so it should start tracking which components or functions depend on it.
  • trigger(): Tells Vue that the value has changed, so it should re-run all dependent watchers or reactivity hooks.

In customRef, you get full control over when reactivity happens:

  • track() is called in the get() method to mark the reactive value as read.
  • trigger() is called in the set() method to notify Vue that the value has changed and any dependent watchers should re-run.

Why This Matters:

  • Fine-grained control over reactivity.
  • Custom behaviors such as debouncing or throttling changes.

toRaw

  • Allows you to access the raw object behind a reactive object.
  • Useful when you need to work with the original, non-proxy object.
import { reactive, toRaw } from "vue"

const state = reactive({ name: "John" })

const rawState = toRaw(state)  // Gets the raw object behind the reactive one

reactive vs ref vs shallowRef

Feature ref reactive shallowRef
Type Primitive (e.g., string, number, boolean) Object, Arrays Object (shallow)
Reactivity Full reactivity, triggers reactivity on value change Full reactivity, triggers on nested changes Only top-level reactivity
Use Case Simple values, such as numbers and strings Complex objects or arrays Large objects where nested reactivity is not needed

✅ Key Takeaways

  • shallowRef: Top-level reactivity for large objects.
  • toRefs: Converts a reactive object into separate refs.
  • readonly: Makes an object or ref immutable.
  • customRef: Creates a custom reactive object with custom get and set behavior.
  • toRaw: Access the original, non-proxied data from a reactive object.
