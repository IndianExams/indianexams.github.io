Async Setup & Suspense

Vue 3 provides powerful tools to handle async logic during component setup and render states gracefully with <Suspense>.

1. Async setup()

  • setup() can be declared as async.
  • If setup() returns a Promise, Vue automatically detects the component as async.
  • Rendering is delayed until the Promise resolves.
  • No explicit configuration is needed — Vue infers it.

Example: Async fetch in setup()

<script setup>
import { ref } from 'vue'

const user = ref(null)

// async setup (implicit, because of await)
const res = await fetch('https://jsonplaceholder.typicode.com/users/1')
user.value = await res.json()
</script>

<template>
  <h2>{{ user.name }}</h2>
  <p>{{ user.email }}</p>
</template>

Note

Since await is used, setup() is async, and Vue suspends rendering until data is ready.

2. <Suspense>

  • <Suspense> provides a wrapper to handle async components.
  • It shows a fallback while waiting and replaces it with the real component when ready.

Example with fallback

<template>
  <Suspense>
    <template #default>
      <UserProfile />
    </template>
    <template #fallback>
      <div>Loading user...</div>
    </template>
  </Suspense>
</template>

3. Error Handling with <Suspense>

  • If an async setup() throws an error, <Suspense> can catch it with the #error slot.
  • Vue passes an Error object to this slot.
  • Use error.message (the standard JS Error.message) to show details.

Example with error slot

<Suspense>
  <template #default>
    <UserProfile />
  </template>

  <template #fallback>
    <div>Loading user...</div>
  </template>

  <template #error="{ error }">
    <div>Error: {{ error.message }}</div>
  </template>
</Suspense>

Simulating an error

<script setup>
throw new Error("Failed to fetch user data")
</script>

This would render:

Error: Failed to fetch user data

4. How Vue Detects Async Components

  • Vue automatically checks whether setup() is async.
  • If it returns a Promise, the component is treated as suspensible.
  • This works for both async setup() functions and implicit async (when using await).

Note

No manual config needed — Vue handles it.

✅ Summary

  • Async setup() makes components suspensible.
  • <Suspense> shows fallback and handles async states.
  • Errors can be caught with the #error slot using error.message.
  • Vue auto-detects async components by checking if setup() returns a Promise.
