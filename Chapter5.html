Vue 3 Composition API Cheatsheet – Chapter 5

Component Communication

1. Props (Parent → Child)

Props are the primary way to pass data from a parent to a child.

<!-- Child.vue -->
<script setup>
defineProps({
  title: String,
  count: Number
})
</script>

<template>
  <h2>{{ title }} - {{ count }}</h2>
</template>
<!-- Parent.vue -->
<Child title="Vue 3" :count="10" />

2. Emits (Child → Parent)

Child components can send events to parents.

<!-- Child.vue -->
<script setup>
const emit = defineEmits(["increment"])

function handleClick() {
  emit("increment")
}
</script>

<template>
  <button @click="handleClick">Add</button>
</template>
<!-- Parent.vue -->
<script setup>
import { ref } from "vue"
import Child from "./Child.vue"

const count = ref(0)
function increment() {
  count.value++
}
</script>

<template>
  <p>Count: {{ count }}</p>
  <Child @increment="increment" />
</template>

3. Slots (Content Distribution)

Slots allow the parent to inject content into the child.

<!-- Child.vue -->
<template>
  <div class="card">
    <slot></slot>
  </div>
</template>
<!-- Parent.vue -->
<Child>
  <p>This is passed via slot!</p>
</Child>

Named slots:

<!-- Child.vue -->
<template>
  <header><slot name="header"></slot></header>
  <main><slot></slot></main>
  <footer><slot name="footer"></slot></footer>
</template>
<!-- Parent.vue -->
<Child>
  <template #header><h1>Header</h1></template>
  <p>Main content here</p>
  <template #footer><small>Footer</small></template>
</Child>

4. Provide / Inject (Dependency Injection)

Provide and Inject are used for sharing state or functions without props or events. It allows communication without prop drilling, especially in deeply nested components.

<!-- GrandParent.vue -->
<script setup>
import { provide, ref } from "vue"

const theme = ref("dark")
// Provide reactive state
provide("theme", theme)
</script>

<template>
  <Parent />
</template>
<!-- Child.vue -->
<script setup>
import { inject } from "vue"
const theme = inject("theme")
</script>

<template>
  <p>Theme: {{ theme }}</p>
</template>

Important:

In cases where the child needs to change the provided state, using readonly() is a good practice to prevent unintended modifications.

Control State Modifications:

<!-- GrandParent.vue -->
<script setup>
import { provide, ref, readonly } from "vue"

const theme = ref("dark")
// Provide readonly state
provide("theme", readonly(theme))
</script>
<!-- Child.vue -->
<script setup>
import { inject } from "vue"
const theme = inject("theme")
// This state is now readonly in the child component
</script>

5. v-model Binding (Two-way Binding)

<!-- Child.vue -->
<script setup>
const props = defineProps(["modelValue"])
const emit = defineEmits(["update:modelValue"])

function updateValue(e) {
  emit("update:modelValue", e.target.value)
}
</script>

<template>
  <input :value="props.modelValue" @input="updateValue" />
</template>
<!-- Parent.vue -->
<script setup>
import { ref } from "vue"
const name = ref("")
</script>

<template>
  <Child v-model="name" />
  <p>Value: {{ name }}</p>
</template>

6. Template Refs (Access Child DOM/Components)

<!-- Parent.vue -->
<script setup>
import { ref, onMounted } from "vue"

const inputRef = ref(null)

onMounted(() => {
  inputRef.value.focus()
})
</script>

<template>
  <input ref="inputRef" />
</template>

7. Exposing Methods (defineExpose)

By default, parent components cannot directly access child methods/variables. defineExpose makes selected properties available.

<!-- Child.vue -->
<script setup>
import { ref } from "vue"

const count = ref(0)
function increment() {
  count.value++
}

// ✅ Expose explicitly
defineExpose({
  count,
  increment
})
</script>

<template>
  <p>Child count: {{ count }}</p>
</template>
<!-- Parent.vue -->
<script setup>
import { ref, onMounted } from "vue"
import Child from "./Child.vue"

const childRef = ref(null)

onMounted(() => {
  childRef.value.increment()
  console.log(childRef.value.count) // accessible
})
</script>

<template>
  <Child ref="childRef" />
</template>

✅ Key Takeaways

  • This chapter covers all main communication patterns in Vue 3 Composition API.
  • props - Parent to child data passing
  • emits - Child to parent event communication
  • slots - Content distribution
  • provide/inject - Dependency injection
  • v-model - Two-way binding
  • refs - Accessing DOM/child components
  • defineExpose - Exposing child methods/variables
